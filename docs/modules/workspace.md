# Рабочая папка проекта

Рабочая папка — это корень проекта, внутри которого приложение ищет Markdown-документы.

## Принципы

- Основная зона документации находится в каталоге `docs/`.
- Приложение может открывать любой `.md` внутри рабочей папки.
- Относительные ссылки разрешаются относительно текущего документа.
- Ссылки на файлы вне рабочей папки считаются недопустимыми для внутренней навигации.

## Структура

- `src/` — исходный код.
- `docs/` — карта проекта, планы, отчёты.
- `scripts/` — автоматизация сборки и запуска.
- `tmp/` — временные файлы.
- `build/debug/` — отладочная сборка.
- `build/release/` — релизная сборка.
